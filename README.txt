* Summarize the project and what problem it was solving.

This program analyzes a list of grocery items sold throughout the day by counting each item and outputting its frequency via a search or a complete list. It is an item-tracking program that tracks how often items are purchased.


* What did you do particularly well?

Creating class methods from the header file and concisely describing their purpose.


* Where could you enhance your code? How would these improvements make your code more efficient, secure, and so on?

Input validation and commenting the purpose of all variables. Input validation helps ensure the program does not crash due to unexpected input.


* Which pieces of the code did you find most challenging to write, and how did you overcome this? What tools or resources are you adding to your support network?

The most challenging parts of the code were the methods for reading input into the map and writing the counts to an external file. W3Schools and GeeksforGeeks helped me with getting the program to do what I wanted.


* What skills from this project will be particularly transferable to other projects or course work?

Skills from this project that will be transferable are the use of objects and keeping the code in main to a minimum. This project was a good exercise in abstraction, as the user interacted with a very simplified interface.


* How did you make this program maintainable, readable, and adaptable?

This program is maintainable, readable, and adaptable because much of the functionality is written in the object. The object can be used in other programs if needed. It is maintainable and readable because the functions and variables have their functionality commented, making it easy to return to in the future.
